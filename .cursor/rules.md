# Cursor ルール（WordPress + Local + Tailwind CSS + three.js）

## 目的（Goals）
- Windows の Local（Local WP）を使い、WordPress をローカルで開発します。
- Tailwind CSS（最初はCDN）と three.js/WebGL を組み込みます。
- 初心者でも安全に進められるよう、変更は小さく、段階的に進めます。

## 言語ポリシー（最重要）
- すべての説明、手順、コメント、注意事項は **日本語** で記述してください。
- 文体は必ず **丁寧語（です・ます調）** にしてください。
- ユーザーが明示的に求めない限り、英語に切り替えないでください。
- コード（PHP/JS/HTML/CSS）内は英語になっても構いませんが、説明文は日本語にしてください。

## 絶対条件（Non-negotiables）
- Laravel は導入しないでください（本プロジェクトでは不要です）。
- 大規模なリファクタリングは避け、Step 1 → Step 5 の順で進めてください。
- WordPress のコアファイルは編集しないでください。
- テーマ編集は可能であれば **子テーマ** を推奨し、難しい場合は必ずバックアップ手順を提示してください。

## コーディング方針（Coding Standards）
- JavaScript は、可能なら ES Modules を利用してください（type="module" / import）。
- グローバル汚染を避け、必要なスコープに閉じてください。
- テーマ配下は `assets/js/` と `assets/css/` を基本にしてください。
- 読み込みは `wp_enqueue_script` / `wp_enqueue_style` を基本にしてください（巨大なインラインスクリプトは避けます）。
- 他プラグインやテーマとの衝突回避のため、ID・クラス・handle名は固有（例：kaori-）にしてください。

## Tailwind CSS 方針
- フェーズ1：CDN を許可し、まず動く状態を最短で作ります。
- フェーズ2（任意）：必要になった場合のみ npm/ビルド運用に移行します（ユーザーが希望した場合のみ）。

## three.js / WebGL 方針
- 最初は最小構成（ジオメトリ＋簡単なマテリアル）で成功体験を優先してください。
- リサイズ対応（Resize）を必ず実装してください。
- `devicePixelRatio` は上限を設け、モバイル負荷を抑えてください。

## 出力フォーマット（必ず守る）
- 「どのファイルを、どのパスに、何を書くか」を明確にし、**フルのコピペ可能な内容**を提示してください。
- WordPress管理画面操作が必要な場合は、クリック経路（例：固定ページ → 新規追加）を必ず示してください。
- Windowsのパス説明はバックスラッシュでも構いませんが、コード上のURL/パスはWordPress流儀（スラッシュ）で書いてください。

## 提案前の安全確認（必ず実施）
- 現在のテーマ名（可能なら子テーマの有無）を確認してください。
- どこに表示するか（HTMLブロック／テーマテンプレート）を確認してください。
- 対象ページ（例：固定ページ「Playground」）を確認してください。
